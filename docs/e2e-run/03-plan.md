## Reference component (the single exemplar — copy-paste-first)

`src/diffusers/schedulers/scheduling_ddim.py` (`DDIMScheduler`). It is the closest prior art and is already
torch-only (`ConfigMixin`/`register_to_config` at `:25`, `SchedulerMixin` at `:28`; no flax in the core
file). The design is a **whole-file copy** of this exemplar with a single rename map applied — no new
abstraction, no mutation of `DDIMScheduler` (authority: `PHILOSOPHY.md` →
`docs/source/en/conceptual/philosophy.md:39-40,47,50` single-file policy; `:101` schedulers must stay
self-contained / must not import from large utils files; `:103` share via `# Copied from`; `:93` copy-adapt
recipe). Redundancy verdict carried from brainstorm: `DDIMScheduler` already supports configurable
`prediction_type`, so an in-place edit (Option 3) is CUT; the task wants a *new variant* = a fresh
copy-adapted file.

**Rename map (apply globally to the copied file, nothing else):** `DDIM` → `DDIMVariant`. This yields class
`DDIMVariantScheduler`, output dataclass `DDIMVariantSchedulerOutput`, file
`scheduling_ddim_variant.py`. (Q1 resolved: names confirmed, matching the r3 working-tree artifacts.)

  name does **NOT** need to be added to the `KarrasDiffusionSchedulers` enum (`scheduling_utils.py:33`) for the
  minimal task — `check_copies`/`make quality` do not require it and no test asserts membership.

## Test Plan (co-equal — designed here, not deferred to implement)

**File:** `tests/schedulers/test_scheduler_ddim_variant.py` (NOT `test_scheduling_*`). **Base class:**
`SchedulerCommonTest` (`tests/schedulers/test_schedulers.py:256`) — the correct base for schedulers; the
common battery drives `check_over_configs` (`:370`), save/load, step shape, etc. **Approach:** mirror
`test_scheduler_ddim.py` 1:1 with the `DDIMScheduler`→`DDIMVariantScheduler` rename — behavior is identical
modulo names (Non-goal: no algorithmic novelty), so the exemplar's assertions transfer exactly.

Header + harness wiring (mirror `test_scheduler_ddim.py:1-10`):
```python
import torch
from diffusers import DDIMVariantScheduler
from .test_schedulers import SchedulerCommonTest

class DDIMVariantSchedulerTest(SchedulerCommonTest):
    scheduler_classes = (DDIMVariantScheduler,)
    forward_default_kwargs = (("eta", 0.0), ("num_inference_steps", 50))
```

Named fast tests to carry over (each asserts via `check_over_configs` / `full_loop`):

- **`test_prediction_type`** (the golden-task lens) — loops `["epsilon", "v_prediction"]` calling
  `check_over_configs(prediction_type=...)` (mirror `:64-66`). This exercises the configurable
  `prediction_type` deliverable (R3). (The `"sample"` branch is covered indirectly via `test_thresholding`'s
  inner loop in the exemplar; keep the exemplar's exact coverage — do not narrow it.)
- **`test_timesteps`**, **`test_steps_offset`** (incl. the `torch.equal([801,601,401,201,1])` assertion),
  **`test_betas`**, **`test_schedules`**, **`test_clip_sample`**, **`test_timestep_spacing`**,
  **`test_rescale_betas_zero_snr`**, **`test_thresholding`** (loops prediction_type × threshold),
  **`test_time_indices`**, plus `get_scheduler_config` and `full_loop` helpers — all carried verbatim from
  `test_scheduler_ddim.py` with only the scheduler-class rename.
- The inherited `SchedulerCommonTest` battery (save/load round-trip, `from_pretrained`, step output shape,
  config consistency) runs automatically via `scheduler_classes`.

**No `@slow` owed** — the work is a name-only variant of an existing fast-tested scheduler; the exemplar test
is entirely fast and the mirrored test stays fast.
